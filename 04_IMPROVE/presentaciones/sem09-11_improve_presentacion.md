## Slide 3: De Analyze a Improve

```
ANALYZE                          IMPROVE
┌──────────────────┐            ┌──────────────────┐
│ Causas raíz      │ ────────→  │ Soluciones que    │
│ VALIDADAS con    │            │ ATACAN las causas │
│ evidencia        │            │ raíz directamente │
└──────────────────┘            └──────────────────┘
```

**Regla crítica:** Cada solución propuesta debe estar vinculada a una causa raíz validada.

> ❌ "Vamos a comprar nuevos equipos" (sin relación con la causa raíz)
> ✅ "La causa raíz es la falta de calibración. Proponemos implementar un programa de calibración mensual"

## Slide 4: Generación de Soluciones

### Técnicas principales:

| Técnica | Descripción | Cuándo usar |
|---------|------------|------------|
| **Brainstorming** | Generación libre de ideas en grupo | Siempre como primer paso |
| **Benchmarking** | Comparar con mejores prácticas de la industria | Cuando hay referentes |
| **SCAMPER** | Sustituir, Combinar, Adaptar, Modificar, Poner en otro uso, Eliminar, Reorganizar | Para modificar proceso existente |
| **TRIZ** | Principios inventivos basados en patrones de innovación | Para soluciones creativas |
| **Lean** | Eliminar desperdicios (mudas) | Cuando hay actividades NVA |

---

## Slide 5: SCAMPER en Detalle

| Letra | Significado | Pregunta Clave |
|-------|------------|---------------|
| **S** | Sustituir | ¿Qué puedo reemplazar? ¿Material, persona, proceso? |
| **C** | Combinar | ¿Puedo unir dos pasos? ¿Combinar funciones? |
| **A** | Adaptar | ¿Qué idea de otra industria puedo adaptar? |
| **M** | Modificar | ¿Puedo cambiar forma, tamaño, secuencia? |
| **P** | Poner en otro uso | ¿Puedo usar este recurso para otra cosa? |
| **E** | Eliminar | ¿Qué pasos o recursos puedo eliminar? |
| **R** | Reorganizar | ¿Puedo reordenar la secuencia? ¿Invertir? |

## Slide 6: Selección de Soluciones - Matriz Pugh

**Pasos:**
1. Definir criterios de evaluación y pesos
2. Seleccionar una solución como referencia (datum)
3. Comparar cada solución contra el datum
4. Sumar puntajes ponderados
5. Seleccionar la ganadora

**Criterios típicos:**
- 🎯 Impacto en el CTQ
- 💰 Costo de implementación
- ⏱️ Tiempo de implementación
- 🔧 Factibilidad técnica
- ⚠️ Riesgo asociado
- 🔄 Sostenibilidad a largo plazo

---

## Slide 7: Mapa del Proceso To-Be

**¿Qué es?** El diagrama del proceso mejorado, después de implementar la solución.

**Comparación:**
```
PROCESO AS-IS                    PROCESO TO-BE
(Estado actual)                  (Estado mejorado)
┌─────────────────┐              ┌─────────────────┐
│ 12 pasos        │              │ 8 pasos         │
│ 45 min ciclo    │     →        │ 30 min ciclo    │
│ 25% defectos    │              │ 10% defectos    │
│ 3 inspecciones  │              │ 1 inspección    │
└─────────────────┘              └─────────────────┘
```

**Elementos a documentar:**
- Pasos eliminados, modificados o agregados
- Nuevos tiempos estimados
- Nuevos puntos de control
- Responsables de cada cambio

## Slide 8: Plan de Implementación

**Elementos esenciales:**
1. **Fases:** Preparación → Piloto → Despliegue → Estabilización
2. **Cronograma:** Diagrama de Gantt con hitos
3. **Recursos:** Humanos, tecnológicos, financieros
4. **Responsables:** Quién hace qué

> **Nota para este curso:** El plan es teórico/propositivo. No se requiere implementación real.

## Slide 9: Análisis Costo-Beneficio

**Indicadores clave:**

| Indicador | Fórmula | ¿Qué indica? |
|-----------|---------|-------------|
| **ROI** | (Beneficio - Costo) / Costo × 100 | Rentabilidad de la inversión |
| **Periodo de recuperación** | Inversión / Ahorro mensual | Meses para recuperar inversión |
| **Relación B/C** | Beneficios / Costos | Si >1 es viable |

**Ejemplo:**
- Inversión: $30,000
- Ahorro mensual estimado: $5,000
- ROI a 1 año: ($60,000 - $30,000) / $30,000 × 100 = **100%**
- Periodo de recuperación: $30,000 / $5,000 = **6 meses**

## Slide 10: Gestión del Cambio

> *"Las personas no se resisten al cambio, se resisten a ser cambiadas"* — Peter Senge

**Modelo de Kotter (simplificado):**
1. Crear urgencia → ¿Por qué cambiar ahora?
2. Formar coalición → ¿Quién apoya el cambio?
3. Comunicar la visión → ¿Qué cambiará y por qué?
4. Empoderar a la acción → Eliminar obstáculos
5. Generar victorias rápidas → Resultados visibles
6. Consolidar → Institucionalizar el cambio
